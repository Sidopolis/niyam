# Solo Dev Template

## Who Is This For?

Independent developers, indie hackers, and solo founders building products alone.
You need guardrails that keep code clean without the overhead of team processes.
No code reviews, no approval gates — just you, your code, and fast iteration.

## What's Included

### Stacks
- **TypeScript** — Catch bugs before runtime, self-documenting types
- **Next.js** — Full-stack in one framework, deploy anywhere
- **Tailwind CSS** — Ship UI fast without context-switching to CSS files

### Role
- **Full-Stack** — You own everything from database to deployment

### Principles
- **Clean Code** — Future-you is your only teammate; be kind to them
- **Performance** — Users don't wait; fast apps convert better

### Workflows
- **Git** — Clean commits, meaningful messages, easy rollbacks
- **Testing** — Enough coverage to deploy with confidence, not more

## Usage

```bash
npx niyam init --template solo-dev
```

This scaffolds a `.niyam/` config with lean rules — enough structure to stay
productive without bureaucratic overhead.

## Example Projects

- Indie SaaS products
- Personal portfolios and blogs
- Side projects and experiments
- Freelance client deliverables
- Weekend hackathon builds

## Philosophy

Minimal process, maximum output. You don't need CI/CD gates when you're the only
committer. You don't need code review workflows when there's no reviewer. But you
do need clean code (because you'll forget what you wrote in 3 weeks) and you do
need tests (because deploying broken code costs you customers). This template
strips everything else away.
